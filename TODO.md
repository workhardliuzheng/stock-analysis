## 一、高价值优化（预期收益较高）

### 1. 超参数自动调优（Optuna/Hyperopt）

- **现状**: ~~XGBoost/LightGBM 参数是手工设定的（max_depth=4, learning_rate=0.05 等）~~
- **方案**: 用 Optuna 在 Walk-Forward 框架内搜索最优参数组合
- **搜索空间**: max_depth, learning_rate, n_estimators, subsample, colsample_bytree, reg_alpha, reg_lambda
- **约束**: 在 Walk-Forward 框架内评估，避免过拟合
- **优化目标**: 方向准确率（direction_accuracy）
- **涉及文件**: `analysis/ml_predictor.py`
- **新增依赖**: `optuna`
- **用法**: `python main.py signal --auto-tune` 或 `python main.py backtest --auto-tune`
- **状态**: [x] 已实现

### 2. 模型持久化与增量预测 (2026-03-30)

- **现状**: 每次信号生成都重新训练模型（1251天数据 × 8个指数 = 4分钟）
- **方案**:
  - 保存模型到本地文件（pickle/joblib格式）
  - 增量预测：只预测最新N天，利用历史预测结果
  - 模型缓存：同指数同配置复用模型
- **预期效果**: 信号执行速度提升10倍（从30s+降到3s以内）
- **涉及文件**: `analysis/ml_predictor.py`, `analysis/index_analyzer.py`
- **难度**: 低
- **状态**: [x] 已实现 (V7-1 - 2026-03-30)
- **回测结果**: 见 BACKTEST_LOG.md

### 3. 标签工程改进

- **现状**: ~~标签为"次日涨跌方向"（pct_chg > 0 → 1, 否则 → 0），日间噪声大~~
- **已实现方案B - 回归目标**: 直接预测收益率大小（回归任务），而非仅方向
  - 模型: XGBClassifier → XGBRegressor
  - 标签: 次日实际收益率 (pct_chg)
  - 评估指标: MAE, RMSE, 方向准确率, IC(信息系数)
  - 信号: 基于预测收益率阈值 (>0.1% → BUY, <-0.1% → SELL)

- **涉及文件**: `analysis/ml_predictor.py`, `analysis/signal_generator.py`
- **状态**: [x] 已实现（方案B）

### 3. 仓位管理优化（2026-03-30 激进优化）

- **现状**: 二元仓位（全仓 100% 或空仓 0%），无中间状态，单一指数持仓
- **方案**:
  - **方案A - 概率仓位**: 根据 ML 概率做比例仓位（概率 70% → 持 70% 仓）
  - **方案B - 分档仓位**: 高置信 BUY 全仓，低置信 BUY 半仓（如概率 >0.7 全仓，0.6-0.7 半仓）
  - **方案C - Kelly 准则**: 根据历史胜率和盈亏比计算最优仓位比例
  - **方案D - 多指数分散** (已实现): 根据各指数预测收益和风险，动态分配仓位
- **新增功能** (2026-03-29):
  - 多指数分散持仓（同一时刻可持仓多个指数）
  - 动态仓位分配（收益加权、风险调整）
  - 风控约束（单指数最大30%，总仓位最大90%）
  - 空仓条件判断（所有指数预测为负时保持空仓）
  - 多指数回测支持（组合总收益率计算）
  - 与现有回测流程整合（PositionBacktester）
- **新增功能** (2026-03-30 激进优化):
  - **差异化阈值**: 大盘指数0.05%，小盘指数0.03%，科创50/中证1000更激进
  - **止损机制**: 单指数-5%止损，组合-15%紧急空仓
  - **仓位动态调整**: 根据市场状态调整仓位权重
- **涉及文件**: `analysis/position_manager.py` (新建), `analysis/backtester.py`
- **难度**: 中高
- **状态**: [x] 已实现 - 第4阶段（多指数分散持仓 + 激进优化）
- **回测结果**: 见 BACKTEST_LOG.md (#8 - 2026-03-30)

### 4. 止损止盈机制

- **现状**: 仅在 SELL 信号时才平仓，持仓期间无风控
- **方案**:
  - 固定止损: 持仓回撤超过 -5% 强制卖出
  - 移动止盈: 盈利超过 +8% 后，回撤 3% 触发止盈
  - 时间止损: 持仓超过 N 天无盈利则平仓
- **涉及文件**: `analysis/backtester.py`
- **难度**: 中等
- **状态**: [x] 已实现 (V6 - 2026-03-30)
  - 单指数-5%止损已实现
  - 组合-15%紧急空仓已实现
  - 详见 BACKTEST_LOG.md #8

## 二、中等价值优化

### 5. 多时间框架联动

- **现状**: 仅使用日线数据
- **方案**: 日线信号 + 周线趋势确认，只有周线也看多时才执行日线 BUY 信号
- **实现思路**:
  - 将日线数据 resample 为周线
  - 计算周线级别均线排列/MACD方向
  - 作为 ML 特征输入，或作为信号融合的过滤条件
- **预期效果**: 减少震荡市中的虚假信号
- **涉及文件**: `analysis/ml_predictor.py`（新增周线特征），`analysis/signal_generator.py`
- **难度**: 中等
- **状态**: [ ] 待实现

### 6. 跨指数特征

- **现状**: 每个指数独立分析，不考虑其他指数信息
- **方案**:
  - 用大盘指数（上证综指、沪深300）的趋势/情绪作为小盘指数的输入特征
  - 大盘/小盘相对强度（如中证500/沪深300比值的趋势）
  - 市场风格轮动信号（大盘股 vs 小盘股强弱对比）
- **涉及文件**: `analysis/ml_predictor.py`, `analysis/index_analyzer.py`
- **难度**: 中高
- **状态**: [ ] 待实现

### 7. 宏观因子

- **现状**: 仅使用技术面+估值数据
- **方案**: 加入宏观经济数据作为 ML 特征
  - 利率（10年期国债收益率）
  - 汇率（美元/人民币）
  - 北向资金净流入
  - 融资融券余额变化
  - Tushare 可提供部分数据（融资融券已有 financing_margin_trading 表）
- **涉及文件**: `analysis/ml_predictor.py`, 可能需要新建数据同步模块
- **难度**: 中高
- **状态**: [ ] 待实现

### 8. 信号融合规则优化

- **现状**: SignalGenerator 的融合规则是固定的 if-else 逻辑
- **方案**:
  - 用 ML 学习最优融合权重（meta-learner / stacking）
  - 根据市场状态动态调整多因子和 ML 的权重比例
  - 当前固定 40% 多因子 + 60% ML，可尝试自适应
- **涉及文件**: `analysis/signal_generator.py`
- **难度**: 中等
- **状态**: [ ] 待实现
